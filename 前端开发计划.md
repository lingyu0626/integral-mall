## 2. 前端统一接口与数据规则（强制）

## 2.1 请求地址规则
- App：`/api/v1/app/**`
- Admin：`/api/v1/admin/**`
- 禁止在页面中硬编码 URL，必须从 `src/api/endpoints.ts` 常量文件引用。

## 2.2 统一响应 JSON

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

分页结构：

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "pageNo": 1,
    "pageSize": 20,
    "total": 0,
    "list": []
  }
}
```

## 2.3 字段一致性规则（与 MySQL 1:1）
1. 返回字段名必须使用 `snake_case`，与表字段同名，不允许私自改为 camelCase。
2. 实体详情/列表项字段必须来自对应数据表字段，禁止新增无来源业务字段。
3. 多表组合详情使用对象分组返回，例如：`order`、`items`、`delivery`、`flows`。
4. 统计类接口如需聚合值，统一放入 `data.ext_json`，避免污染实体字段。
5. 关键写接口必须带 `X-Request-Id`（兑换、积分调整、资产发放等）。
